2 — Sequência: Apostar

O que acontece quando o user clica UP ou DOWN.

sequenceDiagram
  actor User as Apostador
  participant FE as Frontend
  participant BE as Backend
  participant BET as BettingEngine
  participant POOL as LiquidityPool
  participant VAULT as USDC Vault

  User->>FE: Escolhe jogo + window (5min)
  FE->>BE: GET odds atuais
  BE-->>FE: BRA 65% / ARG 30%
  FE-->>User: Mostra gráfico + payout estimado

  User->>FE: Aposta 100 USDC → UP
  FE->>BET: place_bet(UP, 5min, 100)

  BET->>BET: Fee: 100 × 2% = 2 USDC
  BET->>BET: Calcula payout dinâmico
  BET->>POOL: Checa exposição líquida
  POOL-->>BET: net_exposure OK (< 80%)

  BET->>VAULT: Transfer 98 USDC do user
  BET->>POOL: Lock payout no pool
  BET->>BET: Cria Bet PDA (odds_entry: 65%)

  BET-->>FE: ✅ Bet criada — expira em 5min
  FE-->>User: "Aposta aberta: UP 5min"

3 — Sequência: Settlement

Como a aposta é liquidada após o timer expirar.

sequenceDiagram
  participant CRON as Settlement Cron
  participant BE as Backend (DB)
  participant BET as BettingEngine
  participant POOL as LiquidityPool
  participant VAULT as USDC Vault
  actor User as Apostador
  actor LPs as Investidores

  CRON->>CRON: Scan bets onde now > expires_at

  CRON->>BE: GET odds no timestamp de expiração
  BE-->>CRON: BRA 67% (era 65% no entry)

  CRON->>BET: settle_bet(bet_id, odds_expiry: 67%)

  alt User apostou UP e odds subiram (65→67)
    BET->>VAULT: Transfer payout → User
    VAULT-->>User: 💰 Recebe 122.2 USDC
    BET->>POOL: Unlock liquidity restante
  else User apostou UP mas odds caíram (65→63)
    BET->>POOL: Unlock toda liquidity
    BET->>POOL: Distribui fee pros LPs
    POOL-->>LPs: 💰 +1.5 USDC (fee share)
  end

  BET->>BET: Bet status → Won ou Lost

5 — Payout Dinâmico

O payout muda conforme a proporção de apostas UP vs DOWN.

graph LR
  subgraph POOL["Pool de Apostas"]
    UP["🟢 UP: 8,000 USDC\n(80% dos apostadores)"]
    DOWN["🔴 DOWN: 2,000 USDC\n(20% dos apostadores)"]
  end

  UP -- "payout baixo\n1.25x" --> R1["Maioria aposta\n= menos retorno"]
  DOWN -- "payout alto\n5.0x" --> R2["Minoria aposta\n= mais retorno"]
// Aposta de 100 USDC — fee 2%
effective = 100 - (100 × 0.02) = 98 USDC

// Se apostar UP (lado com 8,000):
payout = 98 × (1 + 2,000 / (8,000 + 98)) = 98 × 1.247 = 122 USDC

// Se apostar DOWN (lado com 2,000):
payout = 98 × (1 + 8,000 / (2,000 + 98)) = 98 × 4.81 = 471 USDC
Incentivo natural: quanto mais gente aposta num lado, menor o payout daquele lado — isso atrai apostas pro lado oposto e equilibra o pool automaticamente.

6 — Controle de Risco

Exposição máxima do pool = 80% da liquidity total.

graph TD
  NEW["Nova aposta chega"] --> CALC["Calcula exposição líquida\n|payout_UP - payout_DOWN|"]
  CALC --> CHECK{"net_exposure\n< 80% do pool?"}
  CHECK -- "✅ Sim" --> ACCEPT["Aceita a aposta\nLock liquidity"]
  CHECK -- "❌ Não" --> REJECT["Rejeita a aposta\nPool protegido"]
  REJECT --> WAIT["Aguarda apostas no\nlado oposto equilibrarem"]
// Exemplo: pool = 10,000 USDC
exposure_up   = 7,200   (soma payouts UP)
exposure_down = 3,600   (soma payouts DOWN)

net = |7,200 - 3,600| = 3,600
max = 10,000 × 0.80  = 8,000

3,600 < 8,000 → ✓ ACEITA

7 — Accounts On-chain (PDAs)

Estrutura dos dados gravados na Solana.

classDiagram
  class Match {
    +String match_id
    +Enum sport
    +[Team, Team] teams
    +Enum status: Upcoming|Live|Settled
    +OddsSnapshot current_odds
    +Pubkey pool
    +Vec~MatchEvent~ events
  }

  class LiquidityPool {
    +u64 total_liquidity
    +u64 locked_liquidity
    +u64 exposure_up
    +u64 exposure_down
    +u16 fee_rate
    +Vec~LpShare~ lp_shares
    +Pubkey vault
  }

  class Bet {
    +Pubkey user
    +Enum direction: Up|Down
    +Enum market: Odds|Goal|Cards
    +OddsSnapshot odds_at_entry
    +u64 amount
    +u64 payout
    +Enum window: 1m|5m|10m|20m
    +i64 created_at
    +i64 expires_at
    +Enum status: Open|Won|Lost
  }

  class OddsSnapshot {
    +u16 home
    +u16 away
    +u16 draw
    +i64 timestamp
  }

  Match "1" --> "1" LiquidityPool : has pool
  Match "1" --> "*" Bet : contains bets
  Bet --> OddsSnapshot : odds_at_entry
  Match --> OddsSnapshot : current_odds

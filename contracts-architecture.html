FutOdds — Smart Contracts

Anchor programs on Solana — 3 programs, 4 PDAs, 11 instructions

1 — Programs & CPI Map

Os 3 programs e como se chamam entre si via CPI.

graph LR
  subgraph Programs

    BET["BettingEngine\n─────────────\nplace_bet()\nsettle_bet()\ncancel_bet()"]
    POOL["LiquidityPool\n─────────────\ncreate_pool()\ndeposit()\nwithdraw()\nclaim_fees()"]

  BE["Backend\n(authority)"] -- "signer" --> ORA

  ORA -- "CPI: read odds" --> BET
  BET -- "CPI: lock/unlock" --> POOL
  BET -- "CPI: transfer USDC" --> POOL

  USER["User\n(wallet)"] -- "signer" --> BET
  LP["LP\n(wallet)"] -- "signer" --> POOL
Authority pattern: OracleAdapter e settle_bet só aceitam txs assinadas pela keypair do backend. O user nunca interage direto com o oracle.

2 — PDA Derivation

Seeds usadas para derivar cada account.

PDA Seeds Criado por
Match "match" + match_id OracleAdapter::update_odds (init_if_needed)
LiquidityPool "pool" + match_id LiquidityPool::create_pool
Bet "bet" + match_id + user + nonce BettingEngine::place_bet
Vault "vault" + match_id LiquidityPool::create_pool (token account)

    ──────────────

  LiquidityPool "1" --> "*" LpPosition : tracks each LP
  Match "1" --> "1" LiquidityPool : has pool
  Match "1" --> "*" Bet : has bets
Rent: Match ~175B = 0.0024 SOL, Pool ~210B = 0.0027 SOL, Bet ~135B = 0.0020 SOL. LpPosition separada do Pool pra evitar realloc com Vec.

sequenceDiagram
  participant User
  participant BET as BettingEngine
  participant Match
  participant POOL as LiquidityPool
  participant VAULT as Vault (USDC)

  User->>BET: place_bet(Up, 300s, 100_000_000)

  BET->>Match: read current_odds
  Match-->>BET: home=6500, away=3000

  BET->>BET: fee = amount * fee_rate / 10000
  BET->>BET: effective = amount - fee

  BET->>POOL: read exposure_up, exposure_down, total_liquidity
  BET->>BET: calc payout = eff * (1 + opp_total / (same_total + eff))

  BET->>BET: new_exposure = |exposure_up + payout - exposure_down|
  BET->>BET: require!(new_exposure < total_liq * 80 / 100)

  BET->>VAULT: transfer_checked(user_ata → vault, amount)
  BET->>POOL: locked_liquidity += payout
  BET->>POOL: exposure_up += payout

  BET->>BET: create Bet PDA {odds_entry, payout, expires_at}
  BET-->>User: ✅ Bet created

// Signer: backend authority
pub fn settle_bet(
    ctx: Context<SettleBet>,
    odds_at_expiry_home: u16,
    odds_at_expiry_away: u16,
) -> Result<()>
sequenceDiagram
  participant CRON as Backend Cron
  participant BET as BettingEngine
  participant POOL as LiquidityPool
  participant VAULT as Vault
  participant User

  CRON->>BET: settle_bet(bet_id, expiry_home=6700, expiry_away=2800)

  BET->>BET: load Bet PDA
  BET->>BET: require!(status == Open)
  BET->>BET: require!(now >= expires_at)

  BET->>BET: entry_home=6500, expiry_home=6700
  BET->>BET: movement = expiry_home - entry_home = +200

  alt direction=Up AND movement > 0 → WON
    BET->>BET: status = Won
    BET->>VAULT: transfer payout → user ATA
    VAULT-->>User: 💰 payout USDC
    BET->>POOL: locked -= payout
    BET->>POOL: exposure_up -= payout
  else direction=Up AND movement <= 0 → LOST
    BET->>BET: status = Lost
    BET->>POOL: locked -= payout
    BET->>POOL: exposure_up -= payout
    Note over POOL: USDC stays in vault → LPs profit
  end

  BET->>POOL: distribute fee to LP shares

5 — Validations & Guards

Checagens em cada instruction.

InstructionGuards
update_odds Signer == authority
odds_home + odds_away + odds_draw == 10000
match.status != Settled
place_bet match.status == Live
window ∈ {60, 300, 600, 1200}
amount >= MIN_BET (1 USDC)
amount <= MAX_BET (1000 USDC)
net_exposure + payout < 80% pool
user has enough USDC
settle_bet Signer == authority
bet.status == Open
clock.unix_timestamp >= bet.expires_at
deposit match.status != Settled
amount >= MIN_DEPOSIT (100 USDC)
withdraw shares <= lp_position.shares
redeemable <= total_liquidity - locked_liquidity
cancel_bet bet.status == Open
clock.unix_timestamp < bet.expires_at
signer == bet.user OR signer == authority

6 — Error Codes

pub enum FutOddsError {
    #[msg("Unauthorized signer")]
    Unauthorized,              // 6000
    #[msg("Match is not live")]
    MatchNotLive,              // 6001
    #[msg("Invalid time window")]
    InvalidWindow,             // 6002
    #[msg("Bet amount below minimum")]
    BetTooSmall,               // 6003
    #[msg("Bet amount above maximum")]
    BetTooLarge,               // 6004
    #[msg("Pool exposure limit exceeded")]
    ExposureLimitExceeded,     // 6005
    #[msg("Insufficient liquidity for withdrawal")]
    InsufficientLiquidity,     // 6006
    #[msg("Bet not yet expired")]
    BetNotExpired,             // 6007
    #[msg("Bet already settled")]
    BetAlreadySettled,         // 6008
    #[msg("Odds must sum to 10000")]
    InvalidOddsSum,            // 6009
    #[msg("Insufficient funds")]
    InsufficientFunds,         // 6010
    #[msg("Bet already expired, cannot cancel")]
    BetExpired,                // 6011
}

Cada program isolado: quem assina, o que lê, o que escreve, pra onde vai USDC.
